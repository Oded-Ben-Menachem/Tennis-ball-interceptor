Real-time Tennis Ball Detection & Tracking
A computer vision pipeline designed to detect and track fast-moving tennis balls. This project utilizes a Random Forest classifier and Temporal Analysis to achieve high accuracy even in noisy environments.

🚀 Key Features
Threaded Processing: Separate thread for camera streaming to minimize frame-drop and latency.

Geometric Feature Engineering: Uses a custom feature vector including Solidity, Circularity, and Aspect Ratio to distinguish the ball from complex backgrounds.

Temporal Memory: Implements a deque buffer to analyze motion vectors (displacement and area delta) between frames.

IP Camera Support: Optimized for low-latency streaming via IP-based mobile cameras.

📋 How It Works (Simplified)
The system processes the video in four simple steps to find the ball:

Cleaning the Image: We turn the video to black-and-white and smooth it out (Blur) to remove "visual noise" and tiny dots that aren't important.

Spotting Movement: The system compares the current frame to a "memory" of the empty room. Anything that is different is marked as "moving."

Gluing the Pieces: Fast balls often look like a blurry, broken line. We use "Morphological" filters to glue these pieces back together into one solid shape.

The Brain (AI): A Random Forest model looks at the final shape and its movement path to decide: "Is this a tennis ball or just a shadow?"
